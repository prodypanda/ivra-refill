## 2024-06-28 - Dynamic Feedback and RTL Layout Support

**Learning:** When developing accessible and visually cohesive Flutter apps, it is important to replace default indicators like `CircularProgressIndicator` with skeleton loaders like `CardShimmer` to provide contextual, dynamic feedback. Additionally, when supporting Right-to-Left (RTL) languages like Arabic, physical positioning (e.g., `Alignment.topLeft`, `BorderRadius.only(topLeft: ...)`) breaks layout mirroring. `AlignmentDirectional` and `BorderRadius.vertical` or `BorderRadiusDirectional` are essential. Lastly, in `fl_chart`, properties like `duration` and `curve` must be passed to the top-level Chart widget rather than its Data object to drive implicit animations properly.
**Action:** When implementing new UI elements, always verify RTL compatibility by using Directional widgets. Avoid deprecated `swapAnimation` properties in `fl_chart` and instead use the standardized top-level animation params. Always default to skeleton shimmers over standard spinners for loading complex widgets to enhance perceived performance.
