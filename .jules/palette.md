## 2024-07-02 - Visionary UI Refinements in Reports Screen
**Learning:** We refined the reports screen by leveraging native Riverpod `AsyncValue.isLoading` and `.hasError` properties instead of strictly unwrapping `.valueOrNull` during data load. We also improved accessibility and responsiveness by fixing RTL border-radius issues in charts and report cards, using explicit implicit animations instead of relying on defaults.
**Action:** Always proactively handle `.isLoading` and `.hasError` via `AnimatedSwitcher` and structural loaders (`CardShimmer`) on full-screen async data dependencies. Use `BorderRadius.vertical` or `BorderRadiusDirectional` instead of strictly physical `BorderRadius.only(topLeft/topRight)` to cleanly support Right-to-Left languages automatically. Avoid deprecated implicit properties like `swapAnimationDuration` in modern library versions (like `fl_chart`), setting them natively on widget inputs instead.
