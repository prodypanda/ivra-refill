## 2024-06-25 - Enhance Reports Screen

**Learning:** When developing reports and visual dashboards with data provided via Riverpod, explicitly handling loading and error states for *all* providers is crucial to prevent unhandled UI jumps or errors. Using `AnimatedSwitcher` with skeleton loading (`_ReportsShimmer` composed of `CardShimmer` and `MetricCardShimmer`) provides much smoother UX. Visual data bars (like the one implemented in `_TopList`) make data scanning incredibly fast, and adding animations (`AnimatedContainer` with duration and curves) brings the entire view to life. RTL layout compatibility requires intentional selection of `BorderRadius` elements—using `.vertical()` over physical direction-based properties such as `.only(topLeft: ...)` fixes RTL inversion issues. Finally, unbounded layouts on wide screens can be addressed gracefully with `LayoutBuilder` instead of fixed sizes (like `360`).

**Action:** Consistently use `AnimatedSwitcher` to transition between `AsyncValue` data states and shimmer loaders. When writing custom `fl_chart` representations, configure animation properties correctly directly on the `BarChart` widget rather than properties on its data nodes. Avoid hardcoded layout constraints in responsive apps.
